HireEU

Deploy Guide — No Terminal Needed

4 steps · ~20 minutes · 100% free

1
Upload code to GitHub
Free ~5 min

GitHub stores your code so Vercel can access it. You don't need to know Git — use the website.

  • 1
    Go to github.com → Sign up for a free account
  • 2
    Click the + icon (top right) → New repository
  • 3
    Name it hireeu → set to Private → click Create repository
  • 4
    On the next page, click "uploading an existing file" link
  • 5
    Drag the entire hireeu-vercel folder contents into the upload area (all files and the api/ + public/ folders)
  • 6
    Click Commit changes → done
⚠️ Do NOT upload the .env file if you create one. Your passwords stay on your computer only — you'll enter them in Vercel's dashboard instead.
2
Create a free Neon database
Free forever ~3 min

Neon is a free Postgres database in the cloud. It stores your waitlist email signups.

  • 1
    Go to neon.tech → Sign up free (use your GitHub account)
  • 2
    Click Create Project → name it hireeu → pick any region → Create
  • 3
    On the dashboard, find the Connection String — it looks like:
    postgresql://user:password@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require
  • 4
    Copy that full string and save it in a notepad — you'll need it in Step 3
✅ Neon's free tier gives you 0.5 GB storage — enough for tens of thousands of email signups. No credit card needed.
3
Create a Gmail App Password
Free ~3 min

This lets HireEU send emails through your Gmail without exposing your real password.

  • 1
    Go to myaccount.google.com/security
  • 2
    Make sure 2-Step Verification is turned ON (required)
  • 3
    Search for "App Passwords" in the search bar at top → click it
  • 4
    Type HireEU as the app name → click Create
  • 5
    Google shows a 16-character password like abcd efgh ijkl mnop → copy it (it only shows once)
ℹ️ This App Password is separate from your Gmail login. If you ever want to revoke it, just delete it from the same App Passwords page.
4
Deploy on Vercel
Free forever ~8 min

Vercel hosts your site for free and gives you a public URL instantly.

  • 1
    Go to vercel.com → Sign up with GitHub
  • 2
    Click Add New → Project → select your hireeu GitHub repo
  • 3
    Leave all build settings as-is → don't click Deploy yet
  • 4
    Expand "Environment Variables" and add each row below one by one:
Variable NameValue
DATABASE_URL Your Neon connection string from Step 2 Required
SMTP_HOST smtp.gmail.com Required
SMTP_PORT 587 Required
SMTP_SECURE false Required
SMTP_USER your-gmail@gmail.com Required
SMTP_PASS Your 16-char App Password from Step 3 Required
ADMIN_EMAIL your-gmail@gmail.com (to receive signup alerts) Optional
SITE_URL https://hireeu.vercel.app (update after deploy) Optional
  • 5
    Click Deploy — wait ~60 seconds
  • 6
    Vercel gives you a URL like hireeu.vercel.app — click it to see your live site!
  • 7
    Test it: enter your own email → you should receive the confirmation email within 30 seconds
🎉 Your site is now live! Every time someone signs up, they get an email and you get an admin notification.
★
Optional: Connect a real domain
~$10/yr ~10 min

Want hireeu.app instead of hireeu.vercel.app? Do this after your site is working.

  • 1
    Buy a domain at namecheap.com — search for hireeu.app, hireeu.co, or gethireeu.com
  • 2
    In Vercel: go to your project → Settings → Domains → Add → type your domain
  • 3
    Vercel shows you a CNAME record to copy
  • 4
    In Namecheap: Domain List → Manage → Advanced DNS → add the CNAME record Vercel gave you
  • 5
    Wait 5–30 minutes for DNS to update → your domain is live with free HTTPS ✓
  • 6
    Update the SITE_URL environment variable in Vercel to your new domain, then redeploy

🚀 You're all set

Here's what you'll have when done:

  • Live landing page at yourname.vercel.app (or custom domain)
  • Waitlist signups stored in Neon Postgres database
  • Branded confirmation email sent to every signup
  • Admin alert email sent to you on each signup
  • Live signup counter that updates automatically
  • Free HTTPS certificate (automatic via Vercel)
  • Auto-redeploy when you update code on GitHub
